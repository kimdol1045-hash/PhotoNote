# PhotoNote (가칭) — 사진 캡처·펜 메모·버전관리 웹앱

## 1. 한 줄 정의
**사진 찍고 → 원하는 파일명으로 원본 저장 → 펜으로 그리고 → 저장 시 자동으로 `-1`, `-2`로 분기 → 폴더로 정리**하는 웹앱.

원본은 절대 안 건드림. 편집은 항상 새 파일.

---

## 2. 핵심 기능

### 2.1 촬영 & 저장
- 카메라로 촬영 (또는 갤러리에서 업로드)
- 촬영 직후 **파일명 입력 모달** (default: `IMG_yyyymmdd_HHmmss`)
- 저장 위치: 현재 폴더
- 원본은 immutable — 어디서도 수정 불가

### 2.2 펜 에디터
- 펜 (색상 4종, 굵기 3단)
- 지우개
- Undo / Redo

### 2.3 버전 저장
편집 후 저장 시 자동으로 `-N` 증분:
```
photo.jpg          (원본)
photo-1.jpg        (1차 편집)
photo-2.jpg        (2차 편집)
```
- `-1`을 또 편집해도 다음 번호인 `photo-3.jpg`로 (UI는 평탄 증분, 데이터에만 부모 추적)
- 사진 상세에서 "버전 N개" 토글로 묶어서 보기

### 2.4 폴더 관리
- 폴더 생성·이름 변경·삭제
- 사진 이동 (드래그 또는 선택 후 이동)
- 1단계 폴더 (v1) → 다단계 (v2)

### 2.5 다운로드
- 개별 사진 다운로드
- 폴더 일괄 ZIP 다운로드

## 3. 데이터 모델

```ts
File {
  id: uuid
  name: string              // "photo"
  version: number           // 0 = 원본
  parentId: uuid | null     // 직전 편집의 source
  rootId: uuid              // 원본 파일 id (버전 그룹 키)
  folderId: uuid
  isOriginal: boolean
  blob: Blob                // flatten된 이미지 (편집 시 펜 자국 합쳐서 저장)
  thumbnail: Blob
  createdAt
}

Folder {
  id, name, parentFolderId, createdAt
}
```

**편집은 flatten해서 저장** — 펜 자국은 이미지에 합쳐서 1장으로 보관. 편집 세션 안에서는 undo/redo로 충분하고, 저장 후 펜 자국만 따로 지울 필요는 없다는 판단.

---

## 4. 화면 구성

```
[홈]
 ├─ 폴더 목록
 ├─ 현재 폴더의 사진 그리드 (버전 그룹은 1장으로 묶고 뱃지 "+N")
 └─ + 촬영 버튼

[촬영]
 → 카메라 → 셔터 → [파일명 입력 모달] → 저장 → 홈 복귀

[사진 상세]
 ├─ 큰 미리보기
 ├─ 버전 리스트
 ├─ [편집] [다운로드] [이동] [삭제]

[에디터]
 ├─ 캔버스 + 도구바 (펜·지우개·undo·redo)
 └─ [저장] → 자동 -N 증분, 같은 폴더에 새 파일
   [다른 이름으로 저장] → 새 root로 시작
```

---

## 5. 엣지 케이스

| 상황 | 처리 |
|---|---|
| 같은 이름으로 신규 촬영 | 자동으로 다음 번호 부여 (`photo` 있으면 → `photo-1`) — 묻지 않음 |
| 원본 삭제 | 자식 버전 처리 선택: [모두 삭제 / 자식 유지하고 root 끊기] |
| 큰 이미지 (>10MB) | 자동 리사이즈 옵션 |
| 카메라 권한 거부 | 갤러리 업로드 폴백 |
| IndexedDB 용량 한계 | 사용량 표시, 오래된 썸네일 cleanup |

---

## 6. 기술 스택

- **Framework**: React + Vite + TypeScript
- **Canvas**: Fabric.js
- **카메라**: `getUserMedia`
- **저장**: Dexie.js (IndexedDB)
- **PWA**: Vite PWA plugin (오프라인 동작)
- **다운로드**: JSZip (폴더 일괄)

---

## 7. MVP 범위 (2~3주)

**Week 1**: 프로젝트 셋업, 카메라 캡처 + 파일명 입력 저장, IndexedDB
**Week 2**: 폴더 (생성·이동·삭제), 사진 그리드/상세, 펜 에디터(펜·지우개·undo) + 자동 `-N` 저장
**Week 3**: 다운로드(개별/ZIP), PWA, 폴리싱, 배포
